# OpenClaw Dashboard 移动优先改造方案

## 一、信息架构重排（移动端优先级）

### P0 - 必须首屏可见（核心监控）
| 组件 | 移动端优先级 | 理由 |
|------|------------|------|
| 系统概览 | ⭐⭐⭐ 最高 | 核心健康状态，一眼可知系统是否正常 |
| 健康度评分 | ⭐⭐⭐ 最高 | 快速判断系统状态 |
| 连接状态 | ⭐⭐⭐ 最高 | 网络状态，基础能力 |

### P1 - 次级重要（可折叠/Tab切换）
| 组件 | 移动端优先级 | 交互方案 |
|------|------------|---------|
| Agent列表 | ⭐⭐ 高 | 默认折叠，点击展开/卡片滑动 |
| 当前任务 | ⭐⭐ 高 | 默认显示前3条，更多按钮展开 |
| 模型配额 | ⭐⭐ 高 | 精简视图，点击查看详情 |

### P2 - 可延后加载（Tab/下拉）
| 组件 | 移动端优先级 | 交互方案 |
|------|------------|---------|
| 模型使用统计 | ⭐ 中 | Tab切换查看，图表简化为卡片统计 |
| 实时日志 | ⭐ 中 | 默认隐藏，底部Tab或按钮唤出 |
| 任务历史 | ⭐ 中 | 折叠或独立页面 |
| 通道状态 | ⭐ 中 | 右滑抽屉或底部面板 |

### P3 - 低频操作（侧边菜单/设置）
| 组件 | 移动端优先级 | 交互方案 |
|------|------------|---------|
| 快捷操作 | 低 | 底部浮动按钮或侧滑菜单 |
| 全局搜索 | 低 | 搜索图标点击展开输入框 |

## 二、断点策略

```css
/* 推荐断点体系 */
:root {
  --bp-xs: 360px;   /* 小手机 (iPhone SE, 小米) */
  --bp-sm: 480px;   /* 大手机 (iPhone Pro Max) */
  --bp-md: 768px;   /* 平板竖屏 */
  --bp-lg: 1024px;  /* 平板横屏/小笔记本 */
  --bp-xl: 1366px;  /* 笔记本 */
  --bp-2xl: 1920px; /* 大显示器 */
}

/* 断点使用原则 */
@media (max-width: 479px)     { /* 小手机 - 单列紧凑布局 */ }
@media (min-width: 480px) and (max-width: 767px)  { /* 大手机 - 单列宽松 */ }
@media (min-width: 768px) and (max-width: 1023px) { /* 平板竖屏 - 双列 */ }
@media (min-width: 1024px)                        { /* 桌面 - 多列+侧边栏 */ }
```

### 各断点布局策略

| 断点 | 网格列数 | 侧边栏 | 头部布局 | 卡片交互 |
|------|---------|--------|---------|---------|
| ≤359px | 1列紧凑 | 隐藏(FAB唤出) | 单列堆叠 | 折叠详情 |
| 360-479px | 1列 | 抽屉/底部sheet | 单列堆叠 | 可展开 |
| 480-767px | 1列宽松 | 抽屉/底部sheet | 横向紧凑 | 可展开 |
| 768-1023px | 2列 | 底部横向 | 横向完整 | 完整显示 |
| ≥1024px | 3-4列 | 右侧固定 | 横向完整 | 完整显示 |

## 三、关键组件移动端交互方案

### 3.1 系统概览（P0）

**当前问题：**
- 多个指标横向排列，小屏挤压变形

**移动端方案：**
```
┌─────────────────────────┐
│ 🖥️ 系统概览            │
├─────────────────────────┤
│ ┌─────┐ ┌─────┐ ┌─────┐│
│ │CPU   │ │内存  │ │磁盘  ││
│ │ 45%  │ │ 62%  │ │ 34%  ││
│ └─────┘ └─────┘ └─────┘│
│ ┌─────────────────────┐│
│ │ 更多指标（默认折叠） ││
│ └─────────────────────┘│
└─────────────────────────┘
```

**实现要点：**
- 核心指标（CPU/内存/磁盘）3列小卡片，字体适配
- 次要指标默认折叠，点击"展开更多"
- 图表在小屏简化为迷你柱状图或隐藏

### 3.2 Agent 组织架构（P1）

**当前问题：**
- 树形结构在小屏难以展示
- 搜索框和筛选器占用空间

**移动端方案：**
```
┌─────────────────────────┐
│ 🤖 Agent 组织架构   [🔍]│
├─────────────────────────┤
│ ┌─────────────────────┐│
│ │ 🟢 Agent-001       ││
│ │ 活跃 · 会话:3       ││
│ └─────────────────────┘│
│ ┌─────────────────────┐│
│ │ 🟡 Agent-002       ││
│ │ 空闲 · 会话:0       ││
│ └─────────────────────┘│
│        [加载更多...]    │
└─────────────────────────┘
```

**实现要点：**
- 列表视图替代树形视图
- 搜索框改为图标点击展开
- 筛选改为底部Sheet或Tab
- 每个Agent卡片紧凑：名称+状态+会话数
- 详情点击弹出Modal（已有）

### 3.3 模型使用统计（P2）

**当前问题：**
- 图表在小屏不可读
- 时间范围选择器占用空间

**移动端方案：**
```
┌─────────────────────────┐
│ 📊 模型统计      [7天▼]│
├─────────────────────────┤
│ ┌─────────┐┌─────────┐ │
│ │ GPT-4   ││ Claude  │ │
│ │ 12,345  ││ 8,234   │ │
│ │ ███████ ││ █████   │ │
│ └─────────┘└─────────┘ │
│      [查看图表详情]     │
└─────────────────────────┘
```

**实现要点：**
- 简化为统计卡片+迷你条
- 图表详情按需展开/Modal
- 时间范围改为下拉选择器

### 3.4 实时日志（P2）

**当前问题：**
- 日志面板占用大量空间
- 工具栏在小屏溢出

**移动端方案：**
```
底部Tab栏:
┌─────────────────────────┐
│ [概览] [Agent] [任务] [📝日志]│
└─────────────────────────┘

点击日志Tab后:
┌─────────────────────────┐
│ 📝 实时日志    [筛选▼] │
├─────────────────────────┤
│ 10:23:45 [INFO] 系统启动│
│ 10:23:46 [INFO] 连接建立│
│ 10:23:47 [WARN] 超时重试│
│          [查看更多]      │
└─────────────────────────┘
```

**实现要点：**
- 移动端改为底部Tab项
- 默认显示最近10条日志
- 筛选改为下拉菜单
- 清空等操作放入更多菜单

### 3.5 通知中心

**当前问题：**
- 通知面板从右上角弹出，移动端遮挡内容

**移动端方案：**
- 通知图标放头部右侧
- 点击后全屏底部Sheet展开
- 支持滑动关闭单条通知

### 3.6 右侧边栏（健康度/快捷操作）

**当前实现：** 已有移动端折叠逻辑

**优化要点：**
- FAB按钮位置调整到右下角，避开底部Tab
- 抽屉展开时添加遮罩
- 健康度评分简化为圆形仪表
- 快捷操作改为2x2网格图标

## 四、需要修改的前端文件与风险点

### 4.1 文件修改清单

| 文件 | 修改内容 | 风险等级 |
|------|---------|---------|
| `static/css/style.css` | 添加360/480断点，优化移动端布局 | 🟡 中 |
| `static/css/right-sidebar.css` | 优化移动端侧边栏交互 | 🟢 低 |
| `static/css/layout-improvements.css` | 添加移动端工具类 | 🟢 低 |
| `index.html` | 添加移动端专用结构（底部Tab、折叠按钮） | 🟡 中 |
| `static/js/dashboard.js` | 添加移动端交互逻辑、Tab切换 | 🔴 高 |
| `static/js/ui-enhancements.js` | 添加移动端UI增强（触摸手势） | 🟡 中 |

### 4.2 新增文件

| 文件 | 用途 |
|------|-----|
| `static/css/mobile.css` | 移动端专用样式（可选，也可合并到style.css） |

### 4.3 风险点

1. **CSS兼容性风险**
   - 使用`clamp()`, `min()`, `max()`等CSS函数需确认浏览器支持
   - 触摸事件需测试iOS/Android差异

2. **JavaScript风险**
   - Tab切换需确保状态管理正确
   - 折叠/展开动画需避免性能问题
   - WebSocket重连逻辑需测试移动端网络切换

3. **图表渲染风险**
   - Chart.js在小屏需调整配置
   - 避免canvas尺寸异常导致图表崩溃

4. **触摸交互风险**
   - 避免触摸延迟（使用touch-action优化）
   - 手势与滚动冲突处理

```css
/* 移动端底部Tab栏 */
.mobile-tab-bar {
  display: none;
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  height: 56px;
  background: var(--bg-card);
  border-top: 1px solid var(--border-color);
  z-index: 100;
}

@media (max-width: 767px) {
  .mobile-tab-bar {
    display: flex;
    justify-content: space-around;
    align-items: center;
  }
  
  /* 调整主内容区底部间距 */
  .container {
    padding-bottom: 70px;
  }
}

/* 卡片移动端折叠 */
@media (max-width: 479px) {
  .card .card-content {
    max-height: 0;
    overflow: hidden;
    transition: max-height 0.3s ease;
  }
  
  .card.expanded .card-content {
    max-height: 500px;
  }
}

/* 触摸优化 */
@media (pointer: coarse) {
  .card,
  .btn-small,
  .action-btn {
    min-height: 44px; /* Apple HIG最小触摸目标 */
  }
}
```
